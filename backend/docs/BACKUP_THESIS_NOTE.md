# Нөөцлөлт ба сэргээх төлөвлөгөөний тайлбар

NUM Hospital System нь эмнэлгийн үйлчилгээ, цаг захиалга, өвчтөний бүртгэл, ажилтан, эмчийн үзлэг, үйлчилгээний нөөц, төхөөрөмж, өрөө болон тайлангийн мэдээллийг MongoDB өгөгдлийн санд хадгалдаг. Эдгээр мэдээлэл нь эмнэлгийн үйл ажиллагааны тасралтгүй байдалд чухал тул өгөгдөл алдагдах эрсдэлийг бууруулах зорилгоор үйлдвэрлэлийн орчинд автомат, шифрлэгдсэн нөөцлөлтийн төлөвлөгөө хэрэгжүүлсэн.

Нөөцлөлтийн скрипт нь MongoDB өгөгдлийн сангийн dump-ийг өдөр бүр серверийн цагаар 03:00 цагт үүсгэхээр төлөвлөгдсөн. Dump файл нь `mongodump` хэрэгслээр archive хэлбэрээр үүсэж, gzip шахалттай хадгалагдана. Үүний дараа backup archive нь AWS S3 рүү илгээгдэхээс өмнө сервер дээр client-side public-key encryption ашиглан шифрлэгдэнэ.

Энэхүү шийдэлд AWS S3 дээр зөвхөн шифрлэгдсэн backup файл хадгалагдана. Backup сервер нь зөвхөн public encryption key ашиглана. Private decryption key нь AWS орчинд хадгалагдахгүй бөгөөд offline эсвэл байгууллагын хяналттай, AWS-ээс гаднах аюулгүй орчинд хадгалагдана. Иймээс S3 bucket-д хандах эрх алдагдсан тохиолдолд ч private key байхгүй бол backup-ийн агуулгыг тайлах боломжгүй байна.

S3 lifecycle policy нь `backups/mongodb/` prefix доторх backup объектуудыг 7 хоног хадгалаад түүнээс хуучныг автоматаар устгахаар тохируулагдсан. Энэ нь хадгалалтын зардал болон хэрэгцээгүй хуучин backup хуримтлагдах эрсдэлийг бууруулна.

Энэхүү автомат, шифрлэгдсэн нөөцлөлтийн төлөвлөгөө нь серверийн доголдол, санамсаргүй устгал, deployment-ийн алдаа зэрэг нөхцөлд өгөгдлийг сэргээх боломжийг дэмжинэ. Сэргээх ажиллагаа хийх үед private key-г эрх бүхий оператор гараар өгч, эхлээд staging орчинд туршихыг зөвлөдөг. Энэ баримт бичиг нь хэрэгжүүлсэн скрипт болон тохиргооны хүрээнд тайлбарласан бөгөөд системийн бүрэн disaster recovery бодлогыг орлохгүй.
